# IR Collector (IR_Collect)

[English](README.md) | **中文（繁體）**

可攜式 **Windows live response** 蒐證與檢視工具：單一執行檔，GUI 或 CLI，以 **事實（facts）** 為中心的操作區（時間軸、實體搜尋、調查圖）。**不是**靜默磁碟鑑識平台，也**不是**完整的記憶體鑑識框架。

## 下載

**[最新發行版（ZIP）](https://github.com/9-Security/IR_Collect/releases/latest)** — 內容僅 **`IR_Collect.exe`**，解壓後執行即可。

- **執行環境：** .NET Framework 4.5+（一般 Win7 以上常已具備）。
- **圖形介面：** 連點 `IR_Collect.exe`。
- **CLI 收集：** `IR_Collect.exe -c`（需要完整日誌、原始磁區等情境時，建議以**提高權限**的命令提示字元執行）。

建議將收集**輸出目錄設在外接媒體**，降低對受查主機磁碟的寫入。

## 能做的事

- **蒐集面廣：** 程序、持續性、事件日誌（完整 EVTX + 篩選 CSV）、MFT、USN、登錄匯出、瀏覽器、Prefetch、Jump List、Amcache、ShimCache、SRUM 等。
- **原始跡證 + 正規化事實並存：** Fact Store、時間軸、實體搜尋；另可匯出 HTML 摘要、summary JSON、完整 LOG JSON、調查用 sidecar。
- **橫向移動／身分相關日誌結構化**（RDP、SMB、Kerberos、登入、服務／排程等）利於 pivot 與圖——工具採 **facts-only**，**不**自動幫你下「惡意」判決。
- **覆蓋率／剖析訊息**（如 collection coverage、時間信度、ParserNote）讓「沒收到」比較不會被誤認成「不存在」。
- **記憶體（選用）：** 可在 **Advanced → Settings** 接上**外部**採集與**外部**分析工具；程式**不內建**採集驅動或完整 dump 解析器。

## 不應預期的部分

- **Live response 取捨：** 會寫輸出目錄並留下自身執行痕跡；無法單靠它取代完整離線鑑識或最嚴格的低觸碰流程。
- **高度依賴 Event Log：** 許多高價值敘事前提是日誌**有開、有留**；被清除或未啟用會大幅限縮可見度。
- **記憶體深度：** fileless／純記憶體情境仍高度依賴你接的外部工具與程序。
- **治理：** **沒有內建去識別／修飾**；可選的 AI／上傳能力需由你方制定政策（金鑰、端點、允許清單）。在未設定前，請勿假設適合承載敏感資料。

**摘要：** 適合事件 **前～中期**，用來收斂「哪個帳號、哪個程序、哪個檔、哪個遠端或分享、大約哪段時間」。刻意**不**做成集中式 IR 平台、記憶體分析框架或自動歸因引擎。

**優點（設計上）：** 只產出事實、不代下惡意結論；raw 與正規化並存；EVTX + CSV 雙軌利於複驗與快查；開發流程含 regression／self-test；圖與分鐘級時間軸利於調查敘事。

**限制：** 部分 artifact 現階段僅匯出、未內建解析（如 ShellBags）；資淺分析師仍承擔較多解讀；後續可能含更低干擾模式、補齊非僅依賴日誌的 pivot、修飾設定與上傳治理等。

**一句話：** 適合當 Windows 事件的 **快速蒐證與關聯底座**——強在整理事實與跨來源 pivot；弱在零足跡鑑識、深度記憶體與自動判決。
